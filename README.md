# thread_pool
golang 客户端grpc 线程池实现

## future
- 动态增加、减少线程池的数量。
- 失效线程自动回收
- 动态监控

## 扩容
  如果当前可使用连接数低于5个， 则自动扩容当前线程数量的1倍，直到达到配置中的最大线程数量。
## 缩容
  定时循环查询，如果线程使用率一直低于二分之一以下，则减少空闲数量的一半。最低不能低于配置中的最低线程。
